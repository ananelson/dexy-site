One of the most flexible documentation tools ever.

Dexy was created out of a desire to unify software documentation and scientific document automation, resulting in a tool that is better at both of these than anything that has gone before. With Dexy you have total power and control over what and how you write, with powerful tools to help you manage your data and code.

Dexy lets you to continue to use your favorite documentation tools, while getting more out of them than ever, and being able to combine them in new and powerful ways. With Dexy you can bring your scripting and testing skills into play in your documentation, bringing project automation and integration to new levels.

You can use different filters to show your code in different ways. In the table above, the 'pyg' filter was used to apply syntax highlighting to the raw source code. This filter uses the Pygments API to apply syntax highlighting.Next, we present interpreted output for R, also highlighted using Pygments. Finally, we want to just show the output from our code, so for each language, an output filter is called. For C and Java, this filter first compiles the code. You can view the source for this page to see the different filters.

With Dexy, you can present your code in whatever way makes the most sense for your document. And, if you want to use a different interpreter or compiler, or add support for a new language, it's as simple as writing a new filter. Check out the R, C and Java filters to see how it's done.
