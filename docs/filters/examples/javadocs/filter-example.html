This filter is intended for use with the JSON Doclet, which renders Javadoc data in a reusable JSON format.

To show how this is used, here is a simple Java class:

{{ d['src/hello/helloworld.java|pyg'] }}

After running the Doclet using code like:

{{ d['run-doclet.sh|pyg'] }}

A file called javadoc-data.json is generated:

{{ d['javadoc-data.json|ppjson|pyg'] }}

The javadoc filter takes this information and adds some extra features to it, like syntax highlighting of the java code.

{{ d['javadoc-data.json|javadoc|ppjson|pyg'] }}

You can pull individual elements into your document:

{{ d['javadoc-data.json|javadoc']['packages']['hello']['classes']['helloworld']['methods']['greeting()']['source-html'] }}

Or you can list or iterate over available elements:

{% set helloworld_methods = d['javadoc-data.json|javadoc']['packages']['hello']['classes']['helloworld']['methods'] -%} {% for m in helloworld_methods -%}
{{ m }}

Source:

{{ helloworld_methods[m]['source-html'] }} {% endfor -%}

As you can see from the lack of source code in the main method, the parser which supplements Javadoc info with source code isn't 100% yet. Patches and bug reports very welcome.
