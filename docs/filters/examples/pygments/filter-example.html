Pygments is a syntax highlighting library. Pygments has a large number of supported languages and can handle complex lexing challenges. Pygments is installed along with Dexy so you can use the Pygments filter right away.

You can pass any of these options to the pygments formatter by passing a dict of key/value pairs with the name 'pygments' (see examples below). You need to write the option values in JSON format, main thing to watch out for is that booleans are lower case rather than title case as in Python.

Pygments supports creating anchors for each line of text, letting you link directly to a particular line of code on a page. Dexy's pygments filter automatically configures line anchors using the document's key, this ensures unique anchors if you have code from two different documents on a page. You can set "lineanchors" to something else to override this, set it to an empty string to disable anchors. Anchors are separate from line numbers, you can enable visible line numbers via the "linenos" option, and you can make line numbers clickable links to their respective anchors via the "anchorlinenos" option. Consult the pygments formatter option documentation for details on these and other options.

Pygments uses file extensions to figure out what language's lexer to apply to your code. Most of the time it does a good job, but sometimes this approach doesn't work. You can now specify the lexer to use in your .dexy file. This allows you to use a different file extension than the one Pygments has associated with the lexer, and to highlight languages where there is no designated file extension.

For example, here is a highlighted Sqlite3 console session (from here) saved in a file with a .txt extension:

{{ d['example.txt|pyg'] }}

For HTML and LaTeX output, you can generate stylesheets using the pygmentize command line tool, but there are some shortcuts for this in dexy.

You can now create a pygments stylesheet by specifying an empty virtual file with .css or .sty extension and passing this through the pygments filter (see .dexy config - virtual files begin with an '@). You can specify the pygments style to use. The generated file is linked {{ a['pygments.css|pyg'].hyperlink('here') }}.

You can also include style definitions directly from jinja. You can just type:

{{ "{{ pygments['pastie.css'] }}" }}

or

{{ "<< pygments['pastie.tex'] >>" }}

to insert the style definitions for the 'pastie' style in your document. The 'pygments' object is just a dict and should contain entries for all the pygments styles.

The related IdiopidaeFilter also implements Pygments.
