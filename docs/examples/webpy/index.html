This is an example of using Dexy to document a web application.

We document the source code, seed data (in SQL format), and even take screenshots of the running application, which we can redo at any time.

Here is a handout and some slides containing this demo in PDF format.

First we need to install some software:

{{ d['/install/full-setup-ubuntu.sh|idio']['webpy-installs'] }}

Because we are working with a remote server, we need to create a virtual display which will allow firefox to run and us to take screenshots:

{{ d['/install/build-dexy-site.sh|idio']['virtual-display'] }}

We download and start running a selenium server to allow us to automate a web browser:

{{ d['/install/build-dexy-site.sh|idio']['start-selenium'] }}

Here is the database schema which will back our application:

{{ d['schema.sql|pyg'] }}

We load the schema into a database and start the web.py server:

{{ d['/install/build-dexy-site.sh|idio']['start-webpy'] }}

Now that we are set up, we can run our code and take screenshots:

{{ d['take-screenshots.py|idio']['imports'] }}

We define a helper method to take screenshots:

{{ d['take-screenshots.py|idio']['screencapture'] }}

We launch an instance of Firefox:

{{ d['take-screenshots.py|idio']['start-browser'] }}

Then, we simply load the home page of our web.py app:

{{ d['take-screenshots.py|idio']['load-index-page'] }}
